# Security 

## Introduction

### What is Security?
- Security means "protecting Valuables"

### Threats(보안 위협)
- 컴퓨터 시스템 내의 정보자원에 대해 원하지 않는 결과를 초래할 수 있는 잠재적 가능성, 악의적인 의도, 위협 요소
- Disclosure threats(정보 노출 위협) : 보호되어야 할 정보가 권한이 없는 사용자에게 알려지게 되는 것
- Integrity threats(무결성 위협) : 보호되어야 할 정보가 불법적으로 변경, 생성, 삭제 되는 것
- Denial of Service(서비스 거부 위협) : 권한이 없는 사용자에게 제공되어야 할 서비스의 지연, 방해, 중지 시키는 것
- 이러한 위협으로부터 시스템을 지키는 것이 보안의 목적

### Vulnerability (취약성)
- 보안 위협의 원인을 제공하는 컴퓨터 시스템의 약점
-보안위협을 차단하기 위해 시스템이 가진 취약성들을 찾아 이를 제거 하거나 보완해야 함

### Attack
-악의적 의도를 가진 공격자가 시스템 내에  보호된 정보를 누출, 변조, 파괴하는 행위
- Interruption(방해, 중단)
- Interception(가로채기)
- Modification(변조)
- Fabrication(위조)

### Purpose of Security 
- S = C + I + A
- Confidentiality (기밀성) : 보안의 대상이 되는 컴퓨터 시스템의 자원들에 접근하기 위한 승인을 받는 주체
- Integrity (무결성) : 보안의 대상이 되는 컴퓨터 시스템의 자원들이 승인된 주체들에 의해서만 승인된 방법에 의해서만 변경 가능
- Availability (가용성) : 보안의 대상이 되는 컴퓨터 시스템의 자원들이 승인된 주체들에 의해 접근할 때 방해 받지 말아야 함
